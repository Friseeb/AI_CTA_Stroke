CT Classification Rules

Script: scripts/convert_daylightdicom_to_bids.py  |  Pipeline: DICOM → NIfTI (SLAAOBIDS format)  |  Last updated: 2026-03-16

1. Classification Pipeline Overview

Each DICOM export folder (one study) is classified through 3 sequential steps:

Step 1 — Coverage Gate
Measure anatomical coverage from ImagePositionPatient z-coordinates.
If outside the bounds → disqualify (score = 0).
→
Step 2 — Keyword Scoring
Match DICOM text fields against keywords.
StudyDescription +150 · ProtocolName +80 · BodyPart +50.
Minimum ≥ 50 pts required.
→
Step 3 — Winner
Highest-scoring type wins.
If no type qualifies → study is skipped (not converted).

2. Step 1 — Anatomical Coverage Exclusion (Hard Gate)

coverage_mm = | z_max − z_min | — measured from the ImagePositionPatient DICOM tag, sampling ~20 evenly-spaced files from the largest series in the export.

This metric is scanner-independent and center-independent — it reflects true anatomical extent regardless of PACS naming conventions or language.

If ImagePositionPatient is absent (tag missing), the coverage gate is skipped and classification falls back to keywords only.

Overlap zones — multiple types may survive the coverage gate simultaneously (e.g. a 480 mm scan passes both CTA and TotalBodyCT). Keyword scoring in Step 2 resolves the tie.

3. Step 2 — Keyword Scoring

DICOM FieldPointsNotes
StudyDescription+150Most reliable label — highest weight. Only one keyword needs to match.
ProtocolName+80Medium weight.
BodyPartExamined+50Lowest weight. Sometimes absent or generic.

All keyword matching is case-insensitive substring search. At most one bonus per field per type (first match wins, loop breaks).
Minimum threshold: ≥ 50 pts — a type with no keyword match at all is rejected even if it passed the coverage gate.

4. Per-Type Keyword Lists

CTA — acq-ecta

ProtocolName cta, angio, head/neck, tia +80

Coverage 100–550 mm. Upper bound raised from 450 → 550 mm to accommodate extended head-to-diaphragm CTA protocols. In the 450–550 mm overlap with TotalBodyCT, keyword scoring differentiates them ("cta"/"angio" vs "dissection"/"whole body").

6. Other Pipeline Behaviours

FeatureBehaviour
Idempotent re-runs If the output NIfTI already exists → status = skip_exists. No re-conversion.
Multi-phase export CardiacCT, TotalBodyCT, AbdomenCT: all source series exported as _ph00, _ph01, …
Slice ordering Sorted by z-position (ImagePositionPatient) if ≥70% of slices have the tag, otherwise by InstanceNumber.
Dimension deduplication Mixed-size series (axial + coronal/sagittal reformats) filtered to dominant Rows×Columns pair.
Subject sort order Numeric sort (1, 2, 3 … 299), not alphabetic (1, 10, 100 …).
Output report HTML report (conversion_report_<timestamp>.html) + CSV manifest generated at end of each run.
